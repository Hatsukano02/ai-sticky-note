# 交互规范文档

## 1. 按钮操作规范

### 1.1 按钮类型
- **单按钮设计**: 设备只有一个多功能按钮
- **操作方式**: 单击、双击、长按
- **物理特性**: 轻触开关，6x6x5mm，带触觉反馈

### 1.2 操作时序定义

| 操作类型 | 触发条件 | 防抖动 | 超时处理 |
|---------|---------|--------|----------|
| **单击** | 按下>50ms且<300ms | 50ms | 无 |
| **双击** | 两次单击间隔<400ms | 50ms | 第二次点击超时转为两次单击 |
| **长按** | 按下持续>800ms | 50ms | 最长10秒自动超时 |

### 1.3 各状态下的按钮响应

#### 待机/显示状态
| 操作 | 响应 |
|------|------|
| 单击 | 切换到下一页（循环） |
| 双击 | 切换到上一页（反向循环） |
| 长按(>800ms) | 触发语音输入模式 |

#### 语音输入状态
| 操作 | 响应 |
|------|------|
| 持续按住 | 继续录音 |
| 松开(<10秒) | 结束录音，上传识别 |
| 松开(>10秒) | 强制结束，提示"说话太长" |
| 任何点击 | 忽略（语音模式独占） |

#### 语音处理状态
| 操作 | 响应 |
|------|------|
| 单击 | 取消当前操作，返回显示状态 |
| 双击 | 同单击 |
| 长按 | 忽略 |

#### 同步中状态
| 操作 | 响应 |
|------|------|
| 单击 | 忽略（提示"同步中"） |
| 双击 | 同单击 |
| 长按 | 强制停止同步（危险操作，需确认） |

#### 错误状态
| 操作 | 响应 |
|------|------|
| 单击 | 清除错误，返回正常状态 |
| 双击 | 同单击 |
| 长按 | 重启设备 |

### 1.4 按钮防抖动机制
```cpp
// 伪代码
void button_isr() {
    static uint32_t last_time = 0;
    uint32_t now = millis();
    
    // 50ms防抖动
    if (now - last_time < 50) return;
    last_time = now;
    
    // 记录按下时间
    press_start_time = now;
    is_pressed = true;
}

void button_release_isr() {
    uint32_t duration = millis() - press_start_time;
    is_pressed = false;
    
    if (duration < 300) {
        // 单击/双击检测
        handle_click();
    } else if (duration >= 800 && duration < 10000) {
        // 长按完成
        handle_long_press_end();
    }
}
```

### 1.5 双击检测逻辑
```cpp
void handle_click() {
    static uint32_t last_click_time = 0;
    static int click_count = 0;
    uint32_t now = millis();
    
    if (now - last_click_time < 400) {
        click_count++;
        if (click_count >= 2) {
            // 双击触发
            on_double_click();
            click_count = 0;
            return;
        }
    } else {
        click_count = 1;
    }
    
    last_click_time = now;
    
    // 400ms后如果没有第二次点击，判定为单击
    delay(400);
    if (click_count == 1) {
        on_single_click();
        click_count = 0;
    }
}
```

## 2. 语音交互完整流程

### 2.1 状态流转图

```
[待机状态]
    │
    ▼ 长按>800ms
[长按检测中]
    │
    ▼ 继续按住
[录音中] ──10秒超时──▶ [超时错误]
    │                      │
    ▼ 松开按钮             ▼ 单击
[上传音频] ◀────────── [返回待机]
    │
    ▼
[ASR识别中] ──5秒超时──▶ [识别超时错误]
    │                        │
    ▼ 识别完成               ▼ 单击
[Intent解析] ◀─────────── [返回待机]
    │
    ▼
[执行Tool]
    │
    ├─成功─┬────────────────┐
    │      ▼                │
    │ [播放成功提示]         │
    │      │                │
    │      ▼                │
    │ [更新显示]             │
    │      │                │
    └──────┴──▶ [返回待机]   │
                            │
    ├─失败─┬────────────────┘
    │      ▼
    │ [播放失败提示]
    │      │
    │      ▼
    │ [显示错误信息]
    │      │
    └──────┴──▶ [返回待机]
```

### 2.2 各阶段详细定义

#### 阶段1: 长按检测 (0-800ms)
- **触发**: 用户按下按钮
- **反馈**: 无（避免误触）
- **超时**: 800ms前松开 = 取消操作

#### 阶段2: 长按确认 (800ms)
- **触发**: 按住超过800ms
- **反馈**: 
  - 扬声器播放上升音"叮~"（200ms）
  - LED闪烁1次（100ms）
- **动作**: 启动VAD，开始录音

#### 阶段3: 录音中 (最长10秒)
- **状态**: 持续录音，VAD实时检测
- **反馈**: LED每2秒闪烁1次（表示正在录音）
- **结束条件**:
  - 用户松开按钮（正常结束）
  - VAD检测到静音>2秒（自动结束）
  - 超过10秒（强制结束）

#### 阶段4: 音频上传 (0-3秒)
- **动作**: 通过WebSocket上传音频数据
- **反馈**: 显示"上传中..."动画
- **超时**: 3秒未上传完成 = 网络错误
- **取消**: 单击按钮可取消上传

#### 阶段5: ASR识别 (0-5秒)
- **动作**: 服务器识别语音→文字
- **反馈**: 显示"识别中..."动画
- **超时**: 5秒未返回 = 识别超时
- **取消**: 单击按钮可取消

#### 阶段6: Intent解析 (0-2秒)
- **动作**: LLM解析用户意图，匹配Skill/Tool
- **反馈**: 显示"处理中..."动画
- **结果类型**:
  - 明确意图: 直接执行对应Tool
  - 模糊意图: 请求用户确认
  - 无法识别: 提示"没听懂"

#### 阶段7: Tool执行 (0-3秒)
- **动作**: 执行具体的Skill Tool
- **反馈**: 根据Tool类型显示不同提示
- **结果**:
  - 成功: 播放成功提示音，更新页面
  - 失败: 播放失败提示音，显示错误

#### 阶段8: 返回待机
- **动作**: 回到页面显示状态
- **自动超时**: 如果从长按到完成超过30秒，自动返回

### 2.3 超时与取消机制

| 阶段 | 超时时间 | 超时处理 | 取消方式 |
|------|---------|----------|----------|
| 长按检测 | 800ms | 无操作 | 松开按钮 |
| 录音 | 10秒 | 强制结束录音，提示"说话太长" | 松开按钮 |
| 音频上传 | 3秒 | 提示"网络异常"，重试/取消 | 单击按钮 |
| ASR识别 | 5秒 | 提示"识别超时"，重试/取消 | 单击按钮 |
| Intent解析 | 2秒 | 内部错误，提示"服务繁忙" | 无法取消 |
| Tool执行 | 3秒 | 提示"操作超时" | 无法取消 |
| 整体流程 | 30秒 | 强制返回待机 | 单击按钮 |

### 2.4 打断逻辑

**场景1: 识别过程中想取消**
- 单击按钮 → 立即停止识别 → 播放取消提示音 → 返回待机

**场景2: 正在播报结果时操作**
- 单击/双击: 中断播报，直接执行页面切换
- 长按: 中断播报，开始新的语音输入

**场景3: 正在刷新页面时语音**
- 墨水屏刷新期间(1.5秒) → 忽略按钮操作 → 刷新完成后响应

### 2.5 错误处理

| 错误类型 | 提示音 | 显示信息 | 处理方式 |
|---------|--------|----------|----------|
| 说话太长 | "嘟嘟" | "说话请简短些" | 单击返回 |
| 网络异常 | "嘟嘟嘟" | "网络异常" | 单击返回或长按重试 |
| 识别失败 | "嘟嘟" | "没听清，请重试" | 单击返回或长按重试 |
| 无法识别 | "嘟" | "暂不支持此操作" | 单击返回 |
| 服务繁忙 | "嘟嘟" | "服务器繁忙" | 单击返回 |
| 操作失败 | "嘟嘟嘟" | 具体错误信息 | 单击返回 |

## 3. 状态指示规范

### 3.1 LED指示灯

**硬件规格**: 3mm绿色LED + 3mm红色LED

#### 绿色LED (正常运行)
| 状态 | 闪烁模式 | 含义 |
|------|---------|------|
| 常亮 | 持续 | 设备启动中 |
| 慢闪(2秒周期) | 每2秒闪1次 | 待机/显示状态，正常运行 |
| 快闪(200ms) | 闪1次 | 按钮响应确认 |
| 双闪 | 闪2次 | 页面切换成功 |
| 呼吸灯 | 渐亮渐暗 | 录音中 |

#### 红色LED (异常/充电)
| 状态 | 闪烁模式 | 含义 |
|------|---------|------|
| 常亮 | 持续 | 充电中 |
| 慢闪(2秒周期) | 每2秒闪1次 | 电量低(<20%) |
| 快闪(500ms) | 持续快闪 | 系统错误/崩溃 |
| 闪3次 | 闪3次后停 | 同步失败 |
| 闪5次 | 闪5次后停 | 网络连接失败 |

#### LED组合状态
| 绿 | 红 | 含义 |
|----|----|------|
| 慢闪 | 灭 | 正常运行，电量充足 |
| 慢闪 | 慢闪 | 正常运行，电量低 |
| 快闪 | 灭 | 按钮操作中 |
| 灭 | 常亮 | 充电中 |
| 灭 | 快闪 | 系统错误，需重启 |

### 3.2 扬声器提示音

**硬件规格**: 8Ω 0.5W迷你喇叭

#### 基础提示音
| 场景 | 音效 | 时长 | 音高 |
|------|------|------|------|
| 单击/双击 | "滴" | 50ms | 中音(1kHz) |
| 长按触发 | "叮~" | 200ms | 上升音(800→1200Hz) |
| 录音结束 | "咚" | 100ms | 下降音(1200→800Hz) |
| 操作成功 | "叮" | 150ms | 高音(1500Hz) |
| 操作失败 | "嘟嘟" | 100ms+100ms | 低音(500Hz) |
| 网络错误 | "嘟嘟嘟" | 三连音 | 低音(500Hz) |
| 电量低 | "嘀嘀嘀" | 三短音 | 中音(1kHz) |
| 语音取消 | "嘟" | 50ms | 低音(400Hz) |

#### 语音播报
- **成功操作播报**: "已添加待办" / "切换到天气" / "设置完成"
- **错误播报**: "没听清" / "网络异常" / "操作失败"
- **电量播报**: 电量<20%时，每小时播报一次"电量不足"

#### 播报优先级
1. 错误提示（最高）
2. 操作成功确认
3. 状态提醒（电量等）
4. 背景音（最低）

## 4. 异常场景处理

### 4.1 设备启动流程
```
上电 → 硬件自检 → 加载配置 → 连接WiFi → 同步数据 → 进入待机
   │       │          │          │          │
   ▼       ▼          ▼          ▼          ▼
 失败    失败       失败      重试3次    允许离线
  →红灯快闪  →安全模式  →使用默认  →提示离线 →进入待机
```

### 4.2 死机/卡死处理
- **检测**: 看门狗定时器5秒超时
- **恢复**: 自动重启
- **提示**: 重启后显示"系统恢复"

### 4.3 存储满处理
- **检测**: SPIFFS剩余<10%
- **处理**: 自动清理最旧的日志文件
- **提示**: 显示"存储空间不足"

### 4.4 内存不足处理
- **检测**: 剩余堆内存<20KB
- **处理**: 拒绝新的语音操作，强制同步后重启
- **提示**: 显示"内存不足，重启中"

### 4.5 按钮卡住处理
- **检测**: 按钮按下超过30秒
- **处理**: 视为硬件故障，忽略按钮输入
- **恢复**: 松开按钮后恢复正常

## 7. 附录: 状态机完整定义

```cpp
enum DeviceState {
    // 待机状态
    STATE_IDLE,              // 空闲，显示当前页面
    
    // 按钮操作
    STATE_BUTTON_PRESS,      // 按钮按下，检测中
    STATE_BUTTON_LONG_PRESS, // 长按确认，开始语音
    
    // 语音流程
    STATE_VOICE_RECORDING,   // 录音中
    STATE_VOICE_UPLOADING,   // 上传音频
    STATE_VOICE_RECOGNIZING, // ASR识别中
    STATE_VOICE_EXECUTING,   // 执行Tool
    
    // 同步流程
    STATE_SYNCING,           // 同步数据中
    
    // 特殊状态
    STATE_ERROR,             // 错误状态
    STATE_POWER_ON,          // 开机中
    STATE_SHUTDOWN           // 关机中
};

// 状态转换表
StateTransition transitions[] = {
    // 从IDLE状态
    {STATE_IDLE, EVENT_BUTTON_PRESS, STATE_BUTTON_PRESS},
    {STATE_IDLE, EVENT_SYNC_TRIGGER, STATE_SYNCING},
    {STATE_IDLE, EVENT_ERROR, STATE_ERROR},
    
    // 从BUTTON_PRESS状态
    {STATE_BUTTON_PRESS, EVENT_BUTTON_RELEASE_QUICK, STATE_IDLE},  // 单击执行页面切换
    {STATE_BUTTON_PRESS, EVENT_BUTTON_RELEASE_DOUBLE, STATE_IDLE}, // 双击执行反向切换
    {STATE_BUTTON_PRESS, EVENT_BUTTON_HOLD_800MS, STATE_BUTTON_LONG_PRESS},
    
    // 从BUTTON_LONG_PRESS状态
    {STATE_BUTTON_LONG_PRESS, EVENT_VOICE_START, STATE_VOICE_RECORDING},
    
    // 从VOICE_RECORDING状态
    {STATE_VOICE_RECORDING, EVENT_BUTTON_RELEASE, STATE_VOICE_UPLOADING},
    {STATE_VOICE_RECORDING, EVENT_TIMEOUT_10S, STATE_ERROR},
    
    // 从VOICE_UPLOADING状态
    {STATE_VOICE_UPLOADING, EVENT_UPLOAD_COMPLETE, STATE_VOICE_RECOGNIZING},
    {STATE_VOICE_UPLOADING, EVENT_UPLOAD_FAIL, STATE_ERROR},
    {STATE_VOICE_UPLOADING, EVENT_BUTTON_CLICK, STATE_IDLE}, // 取消
    
    // 从VOICE_RECOGNIZING状态
    {STATE_VOICE_RECOGNIZING, EVENT_RECOGNIZE_COMPLETE, STATE_VOICE_EXECUTING},
    {STATE_VOICE_RECOGNIZING, EVENT_RECOGNIZE_FAIL, STATE_ERROR},
    {STATE_VOICE_RECOGNIZING, EVENT_BUTTON_CLICK, STATE_IDLE}, // 取消
    
    // 从VOICE_EXECUTING状态
    {STATE_VOICE_EXECUTING, EVENT_EXECUTE_SUCCESS, STATE_IDLE},
    {STATE_VOICE_EXECUTING, EVENT_EXECUTE_FAIL, STATE_ERROR},
    
    // 从SYNCING状态
    {STATE_SYNCING, EVENT_SYNC_COMPLETE, STATE_IDLE},
    {STATE_SYNCING, EVENT_SYNC_FAIL, STATE_ERROR},
    
    // 从ERROR状态
    {STATE_ERROR, EVENT_BUTTON_CLICK, STATE_IDLE},
    {STATE_ERROR, EVENT_BUTTON_LONG_PRESS, STATE_IDLE}, // 重启
};
```
