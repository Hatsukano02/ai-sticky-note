# UI显示规范

## 1. 屏幕状态指示

#### 状态栏设计（所有页面通用）
```
┌──────────────────────┐
│● WiFi  🔋85%        │ ← 左上角状态栏
│                      │
│     页面内容...      │
│                      │
└──────────────────────┘
```

**状态栏元素**:
| 图标 | 显示条件 | 含义 |
|------|---------|------|
| ● | WiFi连接 | 在线状态 |
| ○ | WiFi断开 | 离线状态 |
| ⟳ | 同步中 | 正在同步 |
| ✓ | 已同步 | 同步完成 |
| ✗ | 同步失败 | 上次同步失败 |
| 🔋XX% | 始终显示 | 电量百分比 |
| 🔌 | 充电中 | 正在充电 |

#### 全屏状态提示
| 状态 | 显示内容 | 持续时间 | 可取消 |
|------|---------|----------|--------|
| 同步中 | "⟳ 同步中..." | 直到完成 | 否 |
| 同步失败 | "✗ 同步失败，单击重试" | 5秒或单击 | 是 |
| 电量低 | "🔋 电量不足，请及时充电" | 5秒 | 是 |
| 网络错误 | "○ 网络异常" | 5秒或单击 | 是 |
| 语音模式 | "🎤 请说话..." | 直到结束 | 是(单击) |

## 4. 异常场景处理

### 4.1 设备启动流程
```
上电 → 硬件自检 → 加载配置 → 连接WiFi → 同步数据 → 进入待机
   │       │          │          │          │
   ▼       ▼          ▼          ▼          ▼
 失败    失败       失败      重试3次    允许离线
  →红灯快闪  →安全模式  →使用默认  →提示离线 →进入待机
```

### 4.2 死机/卡死处理
- **检测**: 看门狗定时器5秒超时
- **恢复**: 自动重启
- **提示**: 重启后显示"系统恢复"

### 4.3 存储满处理
- **检测**: SPIFFS剩余<10%
- **处理**: 自动清理最旧的日志文件
- **提示**: 显示"存储空间不足"

### 4.4 内存不足处理
- **检测**: 剩余堆内存<20KB
- **处理**: 拒绝新的语音操作，强制同步后重启
- **提示**: 显示"内存不足，重启中"

### 4.5 按钮卡住处理
- **检测**: 按钮按下超过30秒
- **处理**: 视为硬件故障，忽略按钮输入
- **恢复**: 松开按钮后恢复正常

## 6. 加载动画规范

### 6.1 设计原则

墨水屏加载动画需遵循以下原则：

| 原则 | 说明 |
|------|------|
| 简洁优先 | 避免复杂路径，减少局刷残影累积 |
| 黑白分明 | 不使用灰阶，边缘清晰利于墨水颗粒翻转 |
| 时序可控 | 每帧时长固定，确保视觉一致性 |
| 定期清屏 | 周期性全刷，消除局刷累积的残影 |

### 6.2 3×3 点阵动画模式

适用于屏幕中央区域，点阵直径建议 20-30 像素，点间距 ≥10 像素。

**图例说明**：`○` = 白(空心), `●` = 黑, `◐` = 灰阶(过渡)

#### 模式 A：中心呼吸

中心点与四周交替明暗，形成呼吸节奏。5帧循环。

```
帧1      帧2      帧3      帧4      帧5(循环)
○ ○ ○    ● ● ●    ○ ● ○    ● ● ●    ○ ○ ○
○ ● ○    ● ● ●    ● ● ●    ● ● ●    ○ ● ○
○ ○ ○    ● ● ●    ○ ● ○    ● ● ●    ○ ○ ○
  ↑        ↑        ↑        ↑        ↑
仅中心   全亮      中心黑   全黑     回到帧1
        (含中心)   四周亮
```

**时序**：局刷 200ms/帧 → 循环

**帧序说明**：
1. 帧1：仅中心点显示(白底黑圆)
2. 帧2：全屏填满(全黑)
3. 帧3：中心变黑(镂空)，四周保持黑
4. 帧4：全部变黑
5. 帧5：回到帧1状态，形成循环

#### 模式 B：顺时针扫描

沿外圈顺时针逐点移动，中心点常亮作为"核心"。8帧循环。

```
帧1      帧2      帧3      帧4      帧5      帧6      帧7      帧8
● ○ ○    ○ ● ○    ○ ○ ●    ○ ○ ●    ○ ○ ○    ○ ○ ○    ● ○ ○    ● ○ ○
○ ◐ ○    ○ ◐ ○    ○ ◐ ○    ○ ○ ◐    ○ ○ ◐    ○ ◐ ○    ◐ ◐ ○    ◐ ● ○
○ ○ ○    ○ ○ ○    ○ ○ ○    ○ ○ ○    ○ ● ○    ● ○ ○    ○ ○ ○    ○ ○ ○
左上      上中      右上      右下      下中      左下      左中      回到起点
```

**扫描路径**：左上 → 上中 → 右上 → 右下 → 下中 → 左下 → 左中 → 回到起点

**时序**：局刷 150ms/帧 → 每 8 帧全刷一次清残影 → 循环

#### 模式 C：对角线波浪

沿主对角线方向出现，再沿相同方向消退。8帧循环。

```
帧1      帧2      帧3      帧4      帧5      帧6      帧7      帧8
◐ ○ ○    ● ◐ ○    ● ● ◐    ● ● ●    ◐ ● ●    ○ ◐ ●    ○ ○ ◐    ◐ ○ ○
○ ◐ ○    ○ ◐ ○    ● ◐ ○    ● ● ◐    ● ● ◐    ◐ ● ●    ○ ◐ ●    ○ ◐ ○
○ ○ ◐    ○ ○ ◐    ○ ○ ◐    ● ◐ ○    ● ● ◐    ● ● ◐    ◐ ● ●    ○ ○ ◐
出现1    出现2    出现3    填满     消退1    消退2    消退3    回到起点
```

**对角线路径**：
- **出现阶段**（帧1-4）：从左上到右下逐层填充
- **消退阶段**（帧5-7）：同样从左上到右下逐层熄灭
- **循环**（帧8）：回到初始状态

**时序**：局刷 200ms/帧 → 循环

#### 模式 D：随机闪烁

9个点随机点亮/熄灭，模拟星空闪烁效果。8帧循环。

```
帧1      帧2      帧3      帧4      帧5      帧6      帧7      帧8
○ ◐ ○    ◐ ◐ ○    ○ ◐ ○    ◐ ◐ ◐    ○ ◐ ◐    ◐ ◐ ○    ◐ ○ ◐    ○ ○ ○
◐ ○ ◐    ○ ○ ○    ◐ ○ ◐    ○ ○ ◐    ◐ ○ ○    ◐ ○ ◐    ○ ◐ ○    ◐ ○ ◐
○ ○ ○    ◐ ○ ◐    ◐ ○ ◐    ○ ◐ ○    ◐ ◐ ○    ○ ○ ○    ◐ ◐ ○    ◐ ◐ ◐
随机1    随机2    随机3    随机4    随机5    随机6    随机7    随机8
```

**时序**：局刷 150ms/帧 → 每 8 帧全刷一次清残影 → 循环

**随机算法建议**：每次随机选取 3-6 个点点亮，优先使用灰阶过渡帧使变化更平滑。

### 6.3 刷新策略

| 场景 | 推荐策略 | 说明 |
|------|---------|------|
| 短时操作 (<3秒) | 局刷动画 | 快速响应，允许轻微残影 |
| 长时操作 (>3秒) | 局刷+周期性全刷 | 每 8-10 帧全刷一次清残影 |
| 结束过渡 | 全刷黑屏 → 目标页面 | 确保干净切换 |
