# 硬件输入交互规范

## 1. 按钮操作规范

### 1.1 按钮类型
- **单按钮设计**: 设备只有一个多功能按钮
- **操作方式**: 单击、双击、长按
- **物理特性**: 轻触开关，6x6x5mm，带触觉反馈

### 1.2 操作时序定义

| 操作类型 | 触发条件 | 防抖动 | 超时处理 |
|---------|---------|--------|----------|
| **单击** | 按下>50ms且<300ms | 50ms | 无 |
| **双击** | 两次单击间隔<400ms | 50ms | 第二次点击超时转为两次单击 |
| **长按** | 按下持续>800ms | 50ms | 最长10秒自动超时 |

### 1.3 各状态下的按钮响应

#### 待机/显示状态
| 操作 | 响应 |
|------|------|
| 单击 | 切换到下一页（循环） |
| 双击 | 切换到上一页（反向循环） |
| 长按(>800ms) | 触发语音输入模式 |

#### 语音输入状态
| 操作 | 响应 |
|------|------|
| 持续按住 | 继续录音 |
| 松开(<10秒) | 结束录音，上传识别 |
| 松开(>10秒) | 强制结束，提示"说话太长" |
| 任何点击 | 忽略（语音模式独占） |

#### 语音处理状态
| 操作 | 响应 |
|------|------|
| 单击 | 取消当前操作，返回显示状态 |
| 双击 | 同单击 |
| 长按 | 忽略 |

#### 同步中状态
| 操作 | 响应 |
|------|------|
| 单击 | 忽略（提示"同步中"） |
| 双击 | 同单击 |
| 长按 | 强制停止同步（危险操作，需确认） |

#### 错误状态
| 操作 | 响应 |
|------|------|
| 单击 | 清除错误，返回正常状态 |
| 双击 | 同单击 |
| 长按 | 重启设备 |

#### 配对模式（首次启动/重置后）
| 操作 | 响应 |
|------|------|
| 单击 | 无响应（正在等待配对） |
| 双击 | 无响应 |
| 长按 | 无响应 |

#### 深度休眠状态
| 操作 | 响应 |
|------|------|
| 单击 | 唤醒设备，显示当前页面（最新数据） |
| 双击 | 同单击 |
| 长按(>800ms) | 唤醒设备，进入语音输入模式 |

**深度休眠特点**：
- 屏幕保持显示休眠前的内容（墨水屏不耗电）
- WiFi 断开，RTC 定时器运行
- 按键触发 GPIO 中断唤醒
- 唤醒后自动连接 WiFi 并同步数据

**唤醒流程**：
```
按键唤醒 → 连接WiFi → 同步数据 → 处理按键事件 → 刷新显示 → 重置活跃期计时器
```
| 长按 | 无响应 |

**配对模式退出条件**：
1. 配对成功，Registry下发后自动退出
2. 配对码过期（10分钟），屏幕提示"配对码已过期，请重新启动"
3. 用户断电重启，重新生成配对码

#### 设备重置（组合键）

**触发条件**：双击 + 长按（3秒内完成）

```
时间线:
0ms      300ms    400ms    700ms      2700ms   3000ms
 │         │        │        │          │        │
 ▼         ▼        ▼        ▼          ▼        ▼
单击1    松开1   单击2    松开2       长按    长按超过2秒
                 (双击)   (双击确认)   (开始)   (触发重置)
```

**重置流程**：
1. 检测到"双击+长按"组合键
2. 屏幕显示："正在重置设备..."
3. 删除所有配置文件：
   - `/spiffs/config/wifi.json`
   - `/spiffs/config/device.json`
   - `/spiffs/data/registry.json`
4. 屏幕显示："重置完成，即将重启"
5. 延迟1秒后重启
6. 设备进入AP模式（等同于首次启动）

**注意事项**：
- 重置操作不可撤销，所有本地数据将丢失
- 重置后需要重新配网和配对
- 服务器端设备记录保留，管理员可重新绑定

### 1.4 按钮防抖动机制
```cpp
// 伪代码
void button_isr() {
    static uint32_t last_time = 0;
    uint32_t now = millis();
    
    // 50ms防抖动
    if (now - last_time < 50) return;
    last_time = now;
    
    // 记录按下时间
    press_start_time = now;
    is_pressed = true;
}

void button_release_isr() {
    uint32_t duration = millis() - press_start_time;
    is_pressed = false;
    
    if (duration < 300) {
        // 单击/双击检测
        handle_click();
    } else if (duration >= 800 && duration < 10000) {
        // 长按完成
        handle_long_press_end();
    }
}
```

### 1.5 双击检测逻辑
```cpp
void handle_click() {
    static uint32_t last_click_time = 0;
    static int click_count = 0;
    uint32_t now = millis();
    
    if (now - last_click_time < 400) {
        click_count++;
        if (click_count >= 2) {
            // 双击触发
            on_double_click();
            click_count = 0;
            return;
        }
    } else {
        click_count = 1;
    }
    
    last_click_time = now;
    
    // 400ms后如果没有第二次点击，判定为单击
    delay(400);
    if (click_count == 1) {
        on_single_click();
        click_count = 0;
    }
}
```

## 2. 语音交互完整流程

### 2.1 状态流转图

```
[待机状态]
    │
    ▼ 长按>800ms
[长按检测中]
    │
    ▼ 继续按住
[录音中] ──10秒超时──▶ [超时错误]
    │                      │
    ▼ 松开按钮             ▼ 单击
[上传音频] ◀────────── [返回待机]
    │
    ▼
[ASR识别中] ──5秒超时──▶ [识别超时错误]
    │                        │
    ▼ 识别完成               ▼ 单击
[Intent解析] ◀─────────── [返回待机]
    │
    ▼
[执行Tool]
    │
    ├─成功─┬────────────────┐
    │      ▼                │
    │ [播放成功提示]         │
    │      │                │
    │      ▼                │
    │ [更新显示]             │
    │      │                │
    └──────┴──▶ [返回待机]   │
                            │
    ├─失败─┬────────────────┘
    │      ▼
    │ [播放失败提示]
    │      │
    │      ▼
    │ [显示错误信息]
    │      │
    └──────┴──▶ [返回待机]
```

### 2.2 各阶段详细定义

#### 阶段1: 长按检测 (0-800ms)
- **触发**: 用户按下按钮
- **反馈**: 无（避免误触）
- **超时**: 800ms前松开 = 取消操作

#### 阶段2: 长按确认 (800ms)
- **触发**: 按住超过800ms
- **反馈**: 
  - 扬声器播放上升音"叮~"（200ms）
  - LED闪烁1次（100ms）
- **动作**: 启动VAD，开始录音

#### 阶段3: 录音中 (最长10秒)
- **状态**: 持续录音，VAD实时检测
- **反馈**: LED每2秒闪烁1次（表示正在录音）
- **结束条件**:
  - 用户松开按钮（正常结束）
  - VAD检测到静音>2秒（自动结束）
  - 超过10秒（强制结束）

#### 阶段4: 音频上传 (0-3秒)
- **动作**: 通过WebSocket上传音频数据
- **反馈**: 显示"上传中..."动画
- **超时**: 3秒未上传完成 = 网络错误
- **取消**: 单击按钮可取消上传

#### 阶段5: ASR识别 (0-5秒)
- **动作**: 服务器识别语音→文字
- **反馈**: 显示"识别中..."动画
- **超时**: 5秒未返回 = 识别超时
- **取消**: 单击按钮可取消

#### 阶段6: Intent解析 (0-2秒)
- **动作**: LLM解析用户意图，匹配Skill/Tool
- **反馈**: 显示"处理中..."动画
- **结果类型**:
  - 明确意图: 直接执行对应Tool
  - 模糊意图: 请求用户确认
  - 无法识别: 提示"没听懂"

#### 阶段7: Tool执行 (0-3秒)
- **动作**: 执行具体的Skill Tool
- **反馈**: 根据Tool类型显示不同提示
- **结果**:
  - 成功: 播放成功提示音，更新页面
  - 失败: 播放失败提示音，显示错误

#### 阶段8: 返回待机
- **动作**: 回到页面显示状态
- **自动超时**: 如果从长按到完成超过30秒，自动返回

### 2.3 超时与取消机制

| 阶段 | 超时时间 | 超时处理 | 取消方式 |
|------|---------|----------|----------|
| 长按检测 | 800ms | 无操作 | 松开按钮 |
| 录音 | 10秒 | 强制结束录音，提示"说话太长" | 松开按钮 |
| 音频上传 | 3秒 | 提示"网络异常"，重试/取消 | 单击按钮 |
| ASR识别 | 5秒 | 提示"识别超时"，重试/取消 | 单击按钮 |
| Intent解析 | 2秒 | 内部错误，提示"服务繁忙" | 无法取消 |
| Tool执行 | 3秒 | 提示"操作超时" | 无法取消 |
| 整体流程 | 30秒 | 强制返回待机 | 单击按钮 |

### 2.4 打断逻辑

**场景1: 识别过程中想取消**
- 单击按钮 → 立即停止识别 → 播放取消提示音 → 返回待机

**场景2: 正在播报结果时操作**
- 单击/双击: 中断播报，直接执行页面切换
- 长按: 中断播报，开始新的语音输入

**场景3: 正在刷新页面时语音**
- 墨水屏刷新期间(1.5秒) → 忽略按钮操作 → 刷新完成后响应

### 2.5 错误处理

| 错误类型 | 提示音 | 显示信息 | 处理方式 |
|---------|--------|----------|----------|
| 说话太长 | "嘟嘟" | "说话请简短些" | 单击返回 |
| 网络异常 | "嘟嘟嘟" | "网络异常" | 单击返回或长按重试 |
| 识别失败 | "嘟嘟" | "没听清，请重试" | 单击返回或长按重试 |
| 无法识别 | "嘟" | "暂不支持此操作" | 单击返回 |
| 服务繁忙 | "嘟嘟" | "服务器繁忙" | 单击返回 |
| 操作失败 | "嘟嘟嘟" | 具体错误信息 | 单击返回 |
