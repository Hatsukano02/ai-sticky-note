# 交互规范文档

## 1. 按钮操作规范

### 1.1 按钮类型
- **单按钮设计**: 设备只有一个多功能按钮
- **操作方式**: 单击、双击、长按
- **物理特性**: 轻触开关，6x6x5mm，带触觉反馈

### 1.2 操作时序定义

| 操作类型 | 触发条件 | 防抖动 | 超时处理 |
|---------|---------|--------|----------|
| **单击** | 按下>50ms且<300ms | 50ms | 无 |
| **双击** | 两次单击间隔<400ms | 50ms | 第二次点击超时转为两次单击 |
| **长按** | 按下持续>800ms | 50ms | 最长10秒自动超时 |

### 1.3 各状态下的按钮响应

#### 待机/显示状态
| 操作 | 响应 |
|------|------|
| 单击 | 切换到下一页（循环） |
| 双击 | 切换到上一页（反向循环） |
| 长按(>800ms) | 触发语音输入模式 |

#### 语音输入状态
| 操作 | 响应 |
|------|------|
| 持续按住 | 继续录音 |
| 松开(<10秒) | 结束录音，上传识别 |
| 松开(>10秒) | 强制结束，提示"说话太长" |
| 任何点击 | 忽略（语音模式独占） |

#### 语音处理状态
| 操作 | 响应 |
|------|------|
| 单击 | 取消当前操作，返回显示状态 |
| 双击 | 同单击 |
| 长按 | 忽略 |

#### 同步中状态
| 操作 | 响应 |
|------|------|
| 单击 | 忽略（提示"同步中"） |
| 双击 | 同单击 |
| 长按 | 强制停止同步（危险操作，需确认） |

#### 错误状态
| 操作 | 响应 |
|------|------|
| 单击 | 清除错误，返回正常状态 |
| 双击 | 同单击 |
| 长按 | 重启设备 |

#### 配对模式（首次启动/重置后）
| 操作 | 响应 |
|------|------|
| 单击 | 无响应（正在等待配对） |
| 双击 | 无响应 |
| 长按 | 无响应 |

#### 深度休眠状态
| 操作 | 响应 |
|------|------|
| 单击 | 唤醒设备，显示当前页面（最新数据） |
| 双击 | 同单击 |
| 长按(>800ms) | 唤醒设备，进入语音输入模式 |

**深度休眠特点**：
- 屏幕保持显示休眠前的内容（墨水屏不耗电）
- WiFi 断开，RTC 定时器运行
- 按键触发 GPIO 中断唤醒
- 唤醒后自动连接 WiFi 并同步数据

**唤醒流程**：
```
按键唤醒 → 连接WiFi → 同步数据 → 处理按键事件 → 刷新显示 → 重置活跃期计时器
```
| 长按 | 无响应 |

**配对模式退出条件**：
1. 配对成功，Registry下发后自动退出
2. 配对码过期（10分钟），屏幕提示"配对码已过期，请重新启动"
3. 用户断电重启，重新生成配对码

#### 设备重置（组合键）

**触发条件**：双击 + 长按（3秒内完成）

```
时间线:
0ms      300ms    400ms    700ms      2700ms   3000ms
 │         │        │        │          │        │
 ▼         ▼        ▼        ▼          ▼        ▼
单击1    松开1   单击2    松开2       长按    长按超过2秒
                 (双击)   (双击确认)   (开始)   (触发重置)
```

**重置流程**：
1. 检测到"双击+长按"组合键
2. 屏幕显示："正在重置设备..."
3. 删除所有配置文件：
   - `/spiffs/config/wifi.json`
   - `/spiffs/config/device.json`
   - `/spiffs/data/registry.json`
4. 屏幕显示："重置完成，即将重启"
5. 延迟1秒后重启
6. 设备进入AP模式（等同于首次启动）

**注意事项**：
- 重置操作不可撤销，所有本地数据将丢失
- 重置后需要重新配网和配对
- 服务器端设备记录保留，管理员可重新绑定

### 1.4 按钮防抖动机制
```cpp
// 伪代码
void button_isr() {
    static uint32_t last_time = 0;
    uint32_t now = millis();
    
    // 50ms防抖动
    if (now - last_time < 50) return;
    last_time = now;
    
    // 记录按下时间
    press_start_time = now;
    is_pressed = true;
}

void button_release_isr() {
    uint32_t duration = millis() - press_start_time;
    is_pressed = false;
    
    if (duration < 300) {
        // 单击/双击检测
        handle_click();
    } else if (duration >= 800 && duration < 10000) {
        // 长按完成
        handle_long_press_end();
    }
}
```

### 1.5 双击检测逻辑
```cpp
void handle_click() {
    static uint32_t last_click_time = 0;
    static int click_count = 0;
    uint32_t now = millis();
    
    if (now - last_click_time < 400) {
        click_count++;
        if (click_count >= 2) {
            // 双击触发
            on_double_click();
            click_count = 0;
            return;
        }
    } else {
        click_count = 1;
    }
    
    last_click_time = now;
    
    // 400ms后如果没有第二次点击，判定为单击
    delay(400);
    if (click_count == 1) {
        on_single_click();
        click_count = 0;
    }
}
```

```
[上传音频] ◀────────── [返回待机]
    │
    ▼

    │                        │
    ▼ 识别完成               ▼ 单击
[Intent解析] ◀─────────── [返回待机]
    │
    ▼
[执行Tool]
    │
    ├─成功─┬────────────────┐
    │      ▼                │
```

### 2.2 各阶段详细定义

#### 阶段1: 长按检测 (0-800ms)
- **触发**: 用户按下按钮
- **反馈**: 无（避免误触）
- **超时**: 800ms前松开 = 取消操作

#### 阶段2: 长按确认 (800ms)
- **触发**: 按住超过800ms
- **反馈**: 
  - 扬声器播放上升音"叮~"（200ms）
  - LED闪烁1次（100ms）
- **动作**: 启动VAD，开始录音

#### 阶段3: 录音中 (最长10秒)
- **状态**: 持续录音，VAD实时检测
- **反馈**: LED每2秒闪烁1次（表示正在录音）
- **结束条件**:
  - 用户松开按钮（正常结束）
  - VAD检测到静音>2秒（自动结束）
  - 超过10秒（强制结束）

#### 阶段4: 音频上传 (0-3秒)
- **动作**: 通过WebSocket上传音频数据
- **反馈**: 显示"上传中..."动画
- **超时**: 3秒未上传完成 = 网络错误
- **取消**: 单击按钮可取消上传

#### 阶段5: ASR识别 (0-5秒)
- **动作**: 服务器识别语音→文字
- **反馈**: 显示"识别中..."动画
- **超时**: 5秒未返回 = 识别超时
- **取消**: 单击按钮可取消

#### 阶段6: Intent解析 (0-2秒)
- **动作**: LLM解析用户意图，匹配Skill/Tool
- **反馈**: 显示"处理中..."动画
- **结果类型**:
  - 明确意图: 直接执行对应Tool
  - 模糊意图: 请求用户确认
  - 无法识别: 提示"没听懂"

#### 阶段7: Tool执行 (0-3秒)
- **动作**: 执行具体的Skill Tool
- **反馈**: 根据Tool类型显示不同提示
- **结果**:
  - 成功: 播放成功提示音，更新页面
  - 失败: 播放失败提示音，显示错误

#### 阶段8: 返回待机
- **动作**: 回到页面显示状态
- **自动超时**: 如果从长按到完成超过30秒，自动返回

### 2.3 超时与取消机制

| 阶段 | 超时时间 | 超时处理 | 取消方式 |
|------|---------|----------|----------|
| 长按检测 | 800ms | 无操作 | 松开按钮 |
| 录音 | 10秒 | 强制结束录音，提示"说话太长" | 松开按钮 |
| 音频上传 | 3秒 | 提示"网络异常"，重试/取消 | 单击按钮 |
| ASR识别 | 5秒 | 提示"识别超时"，重试/取消 | 单击按钮 |
| Intent解析 | 2秒 | 内部错误，提示"服务繁忙" | 无法取消 |
| Tool执行 | 3秒 | 提示"操作超时" | 无法取消 |
| 整体流程 | 30秒 | 强制返回待机 | 单击按钮 |

### 2.4 打断逻辑

**场景1: 识别过程中想取消**
- 单击按钮 → 立即停止识别 → 播放取消提示音 → 返回待机

**场景2: 正在播报结果时操作**
- 单击/双击: 中断播报，直接执行页面切换
- 长按: 中断播报，开始新的语音输入

**场景3: 正在刷新页面时语音**
- 墨水屏刷新期间(1.5秒) → 忽略按钮操作 → 刷新完成后响应

### 2.5 错误处理

| 错误类型 | 提示音 | 显示信息 | 处理方式 |
|---------|--------|----------|----------|
| 说话太长 | "嘟嘟" | "说话请简短些" | 单击返回 |
| 网络异常 | "嘟嘟嘟" | "网络异常" | 单击返回或长按重试 |
| 识别失败 | "嘟嘟" | "没听清，请重试" | 单击返回或长按重试 |
| 无法识别 | "嘟" | "暂不支持此操作" | 单击返回 |
| 服务繁忙 | "嘟嘟" | "服务器繁忙" | 单击返回 |
| 操作失败 | "嘟嘟嘟" | 具体错误信息 | 单击返回 |

## 3. 状态指示规范

### 3.1 LED指示灯

**硬件规格**: 3mm绿色LED + 3mm红色LED

#### 绿色LED (正常运行)
| 状态 | 闪烁模式 | 含义 |
|------|---------|------|
| 常亮 | 持续 | 设备启动中 |
| 慢闪(2秒周期) | 每2秒闪1次 | 待机/显示状态，正常运行 |
| 快闪(200ms) | 闪1次 | 按钮响应确认 |
| 双闪 | 闪2次 | 页面切换成功 |
| 呼吸灯 | 渐亮渐暗 | 录音中 |

#### 红色LED (异常/充电)
| 状态 | 闪烁模式 | 含义 |
|------|---------|------|
| 常亮 | 持续 | 充电中 |
| 慢闪(2秒周期) | 每2秒闪1次 | 电量低(<20%) |
| 快闪(500ms) | 持续快闪 | 系统错误/崩溃 |
| 闪3次 | 闪3次后停 | 同步失败 |
| 闪5次 | 闪5次后停 | 网络连接失败 |

#### LED组合状态
| 绿 | 红 | 含义 |
|----|----|------|
| 慢闪 | 灭 | 正常运行，电量充足 |
| 慢闪 | 慢闪 | 正常运行，电量低 |
| 快闪 | 灭 | 按钮操作中 |
| 灭 | 常亮 | 充电中 |
| 灭 | 快闪 | 系统错误，需重启 |

### 3.2 扬声器提示音

**硬件规格**: 8Ω 0.5W迷你喇叭

**设计原则**: 纯提示音方案，无语音播报。7个基础提示音覆盖所有交互场景。

#### 基础提示音定义

| 编号 | 音效 | 场景 | 参数 | 描述 |
|------|------|------|------|------|
| S1 | "滴" | 单击、页面切换、WiFi连接 | 50ms, 1kHz | 短促确认音 |
| S2 | "滴滴" | 双击 | 50ms+50ms间隔, 1kHz | 双音确认 |
| S3 | "叮~" | 长按开始录音 | 200ms, 800→1200Hz | 上升音，提示说话 |
| S4 | "咚" | 录音结束、进入休眠 | 100ms, 1200→800Hz | 下降音，表示结束 |
| S5 | "叮叮" | 成功、充电完成 | 150ms+150ms, 1500Hz | 双高音，清脆悦耳 |
| S6 | "嘟" | 失败、错误、低电量 | 300ms, 400Hz | 长低音，沉闷警示 |
| S7 | "叮~" | 唤醒 | 300ms, 600→1200Hz | 渐强上升，温和唤醒 |

#### 音量控制

**自动调节**:
- 白天(8:00-22:00): 100%音量
- 夜间(22:00-8:00): 50%音量

**音量映射**:
```

系统音量  50% → 功放增益 -6dB
```

## 4. 异常场景处理

### 4.1 设备启动流程
```
上电 → 硬件自检 → 加载配置 → 连接WiFi → 同步数据 → 进入待机
   │       │          │          │          │
   ▼       ▼          ▼          ▼          ▼
 失败    失败       失败      重试3次    允许离线
  →红灯快闪  →安全模式  →使用默认  →提示离线 →进入待机
```

### 4.2 死机/卡死处理
- **检测**: 看门狗定时器5秒超时
- **恢复**: 自动重启
- **提示**: 重启后显示"系统恢复"

### 4.3 存储满处理
- **检测**: SPIFFS剩余<10%
- **处理**: 自动清理最旧的日志文件
- **提示**: 显示"存储空间不足"

### 4.4 内存不足处理
- **检测**: 剩余堆内存<20KB
- **处理**: 拒绝新的语音操作，强制同步后重启
- **提示**: 显示"内存不足，重启中"

### 4.5 按钮卡住处理
- **检测**: 按钮按下超过30秒
- **处理**: 视为硬件故障，忽略按钮输入
- **恢复**: 松开按钮后恢复正常

## 8. LED 交互规范

### 8.1 硬件配置

**主状态灯（1个白光LED）**
- 型号：3mm 白光 LED，直插
- 驱动：PWM 控制，1 个 GPIO
- 用途：按钮反馈、语音识别状态、系统状态

**充电灯（1个白光LED）**
- 型号：3mm 白光 LED，直插
- 驱动：数字控制，1 个 GPIO
- 用途：充电状态指示（独立，常亮或呼吸）

**PWM 参数**
```cpp
#define LED_PWM_FREQ      1000    // 1kHz，无频闪
#define LED_PWM_RES       8       // 8-bit 分辨率 (0-255)
#define LED_MIN_BRIGHTNESS 20     // 最小亮度 8%，避免完全熄灭误判
#define LED_MAX_BRIGHTNESS 255    // 最大亮度 100%

// 亮度等级
#define LED_OFF        0
#define LED_DIM        20         // 微亮 8%
#define LED_LOW        50         // 低亮 20%
#define LED_MEDIUM     128        // 中亮 50%
#define LED_HIGH       200        // 高亮 80%
#define LED_FULL       255        // 最亮 100%
```

### 8.2 主状态灯反馈模式

#### 按钮操作反馈

| 操作         | LED 表现                                        | 持续时间 |
| ------------ | ----------------------------------------------- | -------- |
| 单击         | 短闪 100ms（80%亮度）                           | 100ms    |
| 双击         | 双闪 100ms-间隔100ms-100ms                      | 300ms    |
| 长按开始     | 渐亮 500ms（0%→80%）                            | 500ms    |
| 长按保持     | 常亮 80%                                        | 直到松开 |

#### 语音识别状态（含 PWM 音量响应）

**录音阶段（ASR）**
```cpp
// 音量检测 → PWM 映射
void updateVoiceLED(int volumeLevel) {
    // volumeLevel: 0-100 (VAD 检测的音量百分比)
    int brightness;
    
    if (volumeLevel < 20) {
        brightness = LED_DIM;           // 20 (8%)
    } else if (volumeLevel < 60) {
        brightness = map(volumeLevel, 20, 60, LED_DIM, LED_MEDIUM);  // 20-128
    } else {
        brightness = map(volumeLevel, 60, 100, LED_MEDIUM, LED_FULL); // 128-255
    }
    
    ledcWrite(LED_CHANNEL, brightness);
}

// 响应延迟: <50ms
// 效果: 灯随声音"呼吸跳动"
```

**处理阶段（上传+识别+执行）**
```cpp
// 呼吸灯模式
void breathingLED() {
    // 周期: 1.5秒
    // 亮度范围: 30%-70%
    // 效果: 柔和等待感
    static float phase = 0;
    phase += 0.04;  // 1.5秒周期
    if (phase > TWO_PI) phase = 0;
    
    int brightness = LED_LOW + (LED_MEDIUM - LED_LOW) * (0.5 + 0.5 * sin(phase));
    ledcWrite(LED_CHANNEL, brightness);
}
```

**执行结果反馈**

| 结果 | LED 表现                                              |
| ---- | ----------------------------------------------------- |
| 成功 | 快闪两下：■■■ 200ms → 熄灭200ms → ■■■ 200ms           |
| 失败 | 慢闪三下：■■■ 500ms → 熄灭500ms → ■■■ 500ms → 熄灭500ms → ■■■ 500ms |
| 超时 | 快速渐暗：80% → 0% (1秒内)                            |

#### 系统状态指示

| 状态        | LED 表现                      | 含义               |
| ----------- | ----------------------------- | ------------------ |
| 深度休眠    | 熄灭                          | 省电模式           |
| RTC唤醒瞬间 | 渐亮200ms→熄灭                | 系统唤醒（无声）   |
| 同步中      | 慢闪 1Hz（亮500ms/灭500ms）   | 数据同步中         |
| 配网模式    | 双闪-停顿-双闪（2秒周期）     | 等待配对           |
| 低电量      | 慢呼吸 3秒周期（微弱20-40%）  | 电量<20%           |
| WiFi错误    | 快闪 2Hz（持续3次）           | 连接失败           |
| 系统错误    | 常亮 100%（3秒后自动恢复）    | 需要重启           |

### 8.3 充电灯反馈模式

充电灯为独立 LED，仅指示充电状态，与主状态灯分开。

| 充电状态 | LED 表现                    | 设计意图           |
| -------- | --------------------------- | ------------------ |
| 未充电   | 熄灭                        | 不干扰             |
| 充电中   | 慢速呼吸 2秒周期（20%-50%） | 柔和不刺眼         |
| 充满     | 常亮 50%                    | 明确指示，不过亮   |
| 充电异常 | 快闪 3Hz                    | 需检查充电器       |

```cpp
// 充电灯控制（数字 GPIO，非 PWM）
void updateChargeLED(ChargeStatus status) {
    switch(status) {
        case CHARGING:
            // 软件实现呼吸效果
            // 周期: 2秒，亮度: 20%-50%
            break;
        case CHARGED:
            digitalWrite(CHARGE_LED_PIN, HIGH);  // 常亮（外接限流电阻控制亮度）
            break;
        case NOT_CHARGING:
            digitalWrite(CHARGE_LED_PIN, LOW);
            break;
        case CHARGE_ERROR:
            // 快闪 3Hz
            break;
    }
}
```

### 8.5 实现建议

**硬件连接**

```cpp
class LEDController {
```

**注意事项**
1. **最低亮度限制**：设置为 20/255（8%），避免完全熄灭时用户误以为设备关机
2. **夜间友好**：最大亮度控制在 80%（200/255），充电灯充满时仅 50%
3. **响应延迟**：音量响应延迟 <50ms，确保与声音同步
4. **异常恢复**：系统错误状态 3 秒后自动熄灭，避免常亮耗电
